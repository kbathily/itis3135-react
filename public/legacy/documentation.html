Technical Documentation Page

This page summarizes how the React version of the ITIS 3135 site reuses legacy HTML content while keeping navigation consistent through React Router.

Architecture Summary

  • React Router controls page-level navigation
  • A shared site layout provides the course header and footer
  • Legacy HTML files are fetched from the public folder
  • Only each page's main content is injected into the React app

Rendering Flow

  1. The route loads a matching legacy HTML file.
  2. The page title and main content are extracted.
  3. Legacy links are rewritten when they point to React routes.
  4. The cleaned markup is rendered inside the shared layout.

Implementation Notes

Route -> LegacyPage -> fetch HTML -> parse <main> -> rewrite internal links -> render content

This approach keeps the original class assignments usable while gradually moving the site toward a component-based React structure.
